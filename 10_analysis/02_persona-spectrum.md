# 페르소나 스펙트럼

## 핵심 페르소나

### 페르소나 1: 김원장 (45세, 원장)

**역할**
- 도심형 통학 관리형 재수 학원 원장 (지점 3개, 학생 수 약 400명)
- 학원 설립 후 8년차, 교육 경력 20년
- 경영과 교육 모두를 책임지는 소유자 겸 운영자

**목표**
- 학원의 재등록률을 70%에서 85%로 향상시키기
- 학부모 만족도(NPS)를 50에서 70 이상으로 끌어올리기
- 운영 효율화를 통해 인건비를 10% 절감하고 수익성을 개선하기
- 경쟁 학원 대비 차별화된 서비스로 브랜드 가치 제고

**KPI**
- 월 매출, 순이익률
- 재등록률 (전년 대비)
- 학부모 NPS 점수
- 학생당 평균 수강 기간
- 지점별/반별 성과 격차

**고민**
- 매주 금요일마다 학부모 상담 전에 리포트를 준비하는데 원장 본인이 직접 엑셀을 조작하며 5~6시간을 소비한다
- 여러 시스템(출석체크 앱, 모의고사 플랫폼, 결제 시스템)의 데이터를 수동으로 취합해야 해서 실수가 잦고, 데이터 불일치가 발생한다
- 학부모들이 "다른 학원은 더 자세한 리포트를 준다"며 불만을 제기하는 경우가 늘어나고 있다
- 지점 간, 반 간 성과 차이를 객관적으로 분석하고 싶지만 시간이 없어서 직관에 의존하는 경우가 많다
- 리포트 제작을 직원에게 맡기면 품질이 떨어지고, 본인이 하면 전략적 업무에 집중할 시간이 부족하다

**디지털 리터러시 수준**
- 중상: 엑셀, 구글 시트 기본 사용 가능, 스마트폰/태블릿 활용 능숙
- 새로운 SaaS 도구 도입에 대한 거부감은 낮으나, 학습 곡선이 긴 도구는 기피
- 직관적이고 빠르게 결과를 볼 수 있는 도구를 선호

### 페르소나 2: 박학사 (32세, 학사 관리 담당자)

**역할**
- 도심형 통학 관리형 재수 학원의 학사 관리 팀장 (학생 수 약 300명)
- 원장 직속으로 학사 운영 전반을 담당
- 교사들과 학부모 커뮤니케이션을 중간에서 조율

**목표**
- 학부모 상담 시 객관적 데이터를 바탕으로 대화할 수 있도록 하기
- 반별/교사별 성과를 정기적으로 모니터링하여 개선점을 도출하기
- 리포트 제작 시간을 주 15시간에서 주 3시간 이하로 단축하기
- 학생별 맞춤 관리가 필요한 케이스를 조기에 발견하기

**KPI**
- 리포트 제작 소요 시간
- 학부모 상담 만족도 (내부 설문)
- 반별 평균 모의고사 성적 향상률
- 학생별 출석률, 학습 시간 목표 달성률
- 리포트 오류/재작업 빈도

**고민**
- 매주 월요일 아침에 전주 데이터를 취합하여 리포트를 만드는데, 여러 시스템을 오가며 복사-붙여넣기를 반복하다 보니 하루 종일 걸린다
- 엑셀에서 실수로 수식을 잘못 건드려서 잘못된 데이터를 학부모에게 전달한 적이 있어서 신뢰도가 떨어졌다
- 원장이 "이번 주는 왜 리포트가 늦었어?"라고 물어볼 때마다 스트레스를 받는다
- 반별 성과를 비교하고 싶은데, 엑셀로는 차트를 만들기 번거롭고 시간이 오래 걸린다
- 학생 개별 리포트를 만들 때마다 템플릿을 복사하고 데이터를 일일이 입력해야 해서 비효율적이다

**디지털 리터러시 수준**
- 중: 엑셀 중급 수준 (피벗 테이블, 기본 함수 사용 가능), 구글 시트 활용 가능
- 새로운 도구 학습에 대한 의지는 있으나, 업무가 바쁘면 기존 방식에 의존하려는 경향
- 단계별 가이드와 지원이 있으면 빠르게 적응 가능

---

### 페르소나 3: 이관리 (28세, 운영 관리 담당자)

**역할**
- 도심형 통학 관리형 재수 학원의 운영 관리 직원 (학생 수 약 200명)
- 출석 관리, 학습 시간 추적, 모의고사 성적 입력 등 일일 운영 데이터를 관리
- 리포트의 원천 데이터를 입력하고 관리하는 역할

**목표**
- 데이터 입력 오류를 최소화하여 리포트 품질을 높이기
- 반복적인 데이터 입력 작업을 자동화하여 다른 업무에 집중할 시간 확보하기
- 여러 시스템 간 데이터 동기화 문제를 해결하기

**KPI**
- 데이터 입력 정확도
- 데이터 입력 소요 시간
- 시스템 간 데이터 불일치 발생 빈도
- 리포트 재작업 요청 빈도

**고민**
- 출석 앱, 모의고사 플랫폼, 결제 시스템이 각각 다른 곳에 있어서 데이터를 통합하기 어렵다
- 수동으로 데이터를 옮기다 보니 실수가 발생하고, 실수를 발견하면 다시 수정하는데 시간이 더 걸린다
- 리포트를 만들 때마다 "이 데이터 맞아?"라는 확인 요청이 들어와서 업무가 중단된다
- 새로운 시스템을 도입할 때마다 학습해야 하는 부담이 있다
- 데이터 백업이나 버전 관리가 제대로 안 되어서, 실수로 덮어쓰면 복구가 어렵다

**디지털 리터러시 수준**
- 중하: 기본적인 엑셀 사용 가능, 복잡한 함수나 매크로는 사용 불가
- 새로운 도구에 대한 학습 의지는 있으나, 직관적이지 않으면 포기하는 경향
- 단순하고 명확한 인터페이스를 선호

## 공통 Pain Point

### 1. 데이터 소스 분산과 수동 통합의 비효율
- 여러 시스템(출석, LMS, 모의고사, 결제)에 데이터가 흩어져 있어 수동으로 취합해야 함
- 복사-붙여넣기 과정에서 실수 발생, 데이터 불일치 문제 지속

### 2. 리포트 제작 시간의 과다 소비
- 주 10~20시간을 리포트 제작에 소비
- 반복적이고 단순한 작업에 인력이 묶여 전략적 업무에 집중할 수 없음

### 3. 데이터 정확성에 대한 불안
- 수동 입력 과정에서 발생하는 오류로 인한 신뢰도 저하
- 학부모나 원장에게 잘못된 데이터를 전달할 위험에 대한 스트레스

### 4. 비교 분석의 어려움
- 지점별, 반별, 학생별 성과를 비교하고 시각화하는 것이 번거로움
- 객관적 데이터 기반 의사결정이 어려움

---

## 차별화된 Pain Point

### 김원장 (원장)
- **전략적 관점의 Pain**: 리포트 제작에 시간을 쓰느라 학원 경영 전략, 마케팅, 브랜딩 등에 집중할 시간이 부족
- **재무적 관점의 Pain**: 리포트 제작을 위한 인건비가 부담, ROI를 명확히 측정하고 싶음
- **경쟁력 관점의 Pain**: 경쟁 학원 대비 차별화된 리포트 서비스를 제공하고 싶으나 자원 부족

### 박학사 (학사 관리 담당자)
- **업무 효율성 관점의 Pain**: 리포트 제작으로 인한 업무 과부하, 다른 학사 관리 업무에 집중하고 싶음
- **품질 관리 관점의 Pain**: 리포트 오류로 인한 신뢰도 저하, 일관된 품질의 리포트를 만들고 싶음
- **커뮤니케이션 관점의 Pain**: 학부모 상담 시 객관적 데이터를 빠르게 참조하고 싶음

### 이관리 (운영 관리 담당자)
- **데이터 관리 관점의 Pain**: 여러 시스템 간 데이터 동기화 문제, 중복 입력의 번거로움
- **오류 방지 관점의 Pain**: 수동 입력 과정에서의 실수, 데이터 검증 메커니즘의 부재
- **학습 부담 관점의 Pain**: 새로운 도구 학습에 대한 부담, 직관적이지 않은 도구에 대한 거부감
