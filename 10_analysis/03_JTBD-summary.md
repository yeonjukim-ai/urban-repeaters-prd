# JTBD 인터뷰 결과 요약

## 핵심 Job to be Done 목록

1. **학부모 상담 전에 학생의 종합 성과 리포트를 빠르게 준비하고 싶다**
2. **지점별/반별 성과를 비교하여 개선점을 도출하고 싶다**
3. **학부모에게 정기적으로 전문적이고 일관된 리포트를 자동으로 전달하고 싶다**
4. **여러 시스템의 데이터를 한 곳에서 통합하여 확인하고 싶다**
5. **학생별 맞춤 관리가 필요한 케이스를 조기에 발견하고 싶다**
6. **원장/경영진에게 객관적 데이터 기반 경영 리포트를 제공하고 싶다**
7. **모의고사 성적 추이와 학습 활동 데이터를 연계하여 분석하고 싶다**

## Job 1: 학부모 상담 전에 학생의 종합 성과 리포트를 빠르게 준비하고 싶다

### 상황 (Context)
- 학부모 상담이 예약되어 있고, 상담 1~2시간 전에 해당 학생의 최신 성과 데이터를 확인해야 함
- 학생의 출석률, 학습 시간, 모의고사 성적, 과제 완료도 등 여러 지표를 종합적으로 봐야 함
- 상담 시 학부모가 "우리 아이는 어떻게 하고 있어요?"라고 물어볼 때 객관적 데이터로 답변해야 함

### Pain
- 여러 시스템(출석 앱, LMS, 모의고사 플랫폼)을 오가며 데이터를 수집해야 해서 시간이 오래 걸림
- 엑셀에서 학생별 리포트 템플릿을 복사하고 데이터를 일일이 입력하는 과정이 번거로움
- 상담 직전에 급하게 리포트를 만들다 보니 실수가 발생하거나, 데이터가 최신이 아닐 수 있음
- 리포트 형식이 매번 달라져서 일관성이 떨어짐

### Desired Outcome
- 상담 30분 전에 버튼 하나로 해당 학생의 최신 종합 리포트가 자동 생성됨
- 리포트에 출석률, 학습 시간, 모의고사 성적 추이, 과제 완료도, 반 평균 대비 위치 등이 한눈에 보임
- 리포트가 전문적이고 시각적으로 보기 좋아서 학부모에게 바로 공유 가능
- 리포트에 인사이트(예: "최근 학습 시간이 감소 추세입니다")가 포함되어 상담 시 대화의 시작점이 됨

### 현재 사용하는 대체 솔루션
- 엑셀 템플릿 + 수동 데이터 입력 (가장 일반적)
- 구글 시트에 데이터를 모아두고 필요할 때마다 복사-붙여넣기
- 각 시스템을 따로 열어서 스크린샷을 찍어서 보여주기
- 리포트 제작을 외부 업체에 아웃소싱 (비용 부담, 시간 지연)

---

## Job 2: 지점별/반별 성과를 비교하여 개선점을 도출하고 싶다

### 상황 (Context)
- 원장이나 학사 관리자가 주간/월간 회의에서 지점별, 반별 성과를 리뷰해야 함
- "A 지점은 왜 재등록률이 높고, B 지점은 낮을까?" 같은 질문에 데이터로 답변해야 함
- 반별 모의고사 평균 성적, 출석률, 학습 시간 등을 비교하여 개선이 필요한 반을 파악해야 함

### Pain
- 각 지점/반의 데이터를 수동으로 취합하고 비교하는데 시간이 많이 걸림
- 엑셀로 차트를 만들면 형식이 매번 달라지고, 시각화가 어려움
- 데이터가 여러 시스템에 흩어져 있어서 통합 분석이 어려움
- 비교 분석 결과를 경영진에게 제시할 때 전문적으로 보이지 않음

### Desired Outcome
- 대시보드에서 지점별/반별 핵심 지표(재등록률, 모의고사 평균, 출석률 등)를 한눈에 비교 가능
- 시각적 차트(바 차트, 라인 차트)로 성과 격차를 직관적으로 파악 가능
- "이번 달 B 지점의 출석률이 평균 대비 15% 낮습니다" 같은 인사이트가 자동으로 제시됨
- 비교 리포트를 PDF로 내보내어 회의 자료로 활용 가능

### 현재 사용하는 대체 솔루션
- 엑셀 피벗 테이블로 데이터 집계 후 수동으로 차트 작성
- 구글 시트에서 각 지점/반 데이터를 별도 시트로 관리하고 비교
- 각 지점 관리자에게 데이터를 요청받아서 취합 (시간 지연)
- 직관에 의존한 정성적 판단 (데이터 기반 아님)

---

## Job 3: 학부모에게 정기적으로 전문적이고 일관된 리포트를 자동으로 전달하고 싶다

### 상황 (Context)
- 학부모들이 자녀의 학습 현황을 정기적으로 알고 싶어 함 (주 1회 또는 월 2회)
- 학원의 전문성을 보여주기 위해 리포트가 일관되고 전문적으로 보여야 함
- 리포트 전달이 누락되면 학부모 불만이 발생함

### Pain
- 매주/매월 리포트를 일일이 만들고 이메일로 전송하는데 시간이 많이 걸림
- 리포트 형식이 매번 달라져서 일관성이 떨어지고, 전문성이 떨어져 보임
- 학생 수가 많으면 리포트 제작과 전송에 하루 종일 걸림
- 리포트 전송을 깜빡하면 학부모가 "이번 주 리포트는 없나요?"라고 문의함

### Desired Outcome
- 리포트 전송 일정을 설정하면 자동으로 리포트가 생성되고 학부모에게 이메일/SMS로 전송됨
- 리포트 형식이 일관되고 전문적이어서 학원의 브랜드 이미지를 높임
- 리포트에 개인화된 메시지(예: "김철수 학생은 이번 주 학습 시간 목표를 달성했습니다!")가 포함됨
- 리포트 전송 내역과 열람 여부를 추적할 수 있어서 커뮤니케이션 효과를 측정 가능

### 현재 사용하는 대체 솔루션
- 매주 금요일 오후에 엑셀로 리포트를 만들고 이메일로 일괄 전송 (수동)
- 카카오톡 단체방에 스크린샷을 올리기 (비전문적)
- 리포트 전송을 외부 업체에 아웃소싱 (비용 부담)
- 리포트 전송을 생략하거나 간헐적으로만 전송 (학부모 불만)

---

## Job 4: 여러 시스템의 데이터를 한 곳에서 통합하여 확인하고 싶다

### 상황 (Context)
- 출석 관리 앱, LMS(학습 관리 시스템), 모의고사 플랫폼, 결제 시스템 등 여러 시스템을 사용 중
- 각 시스템마다 로그인하고 데이터를 확인하는 것이 번거로움
- 여러 시스템의 데이터를 연계하여 분석하고 싶지만 통합이 어려움

### Pain
- 각 시스템마다 다른 인터페이스와 로그인 정보를 기억해야 함
- 데이터를 비교하거나 연계 분석하려면 여러 시스템을 오가며 수동으로 데이터를 옮겨야 함
- 시스템 간 데이터 불일치가 발생 (예: 출석 앱에는 출석했는데 LMS에는 학습 기록이 없음)
- 새로운 시스템을 도입할 때마다 통합이 어려워서 데이터가 더 분산됨

### Desired Outcome
- 하나의 대시보드에서 모든 시스템의 데이터를 통합하여 확인 가능
- 출석률, 학습 시간, 모의고사 성적, 결제 현황 등이 한 화면에 표시됨
- 시스템 간 데이터 불일치를 자동으로 감지하고 알림
- 새로운 시스템을 추가할 때 API 연동으로 자동 통합 가능

### 현재 사용하는 대체 솔루션
- 각 시스템을 브라우저 탭으로 열어두고 수동으로 전환하며 확인
- 엑셀/구글 시트에 각 시스템 데이터를 수동으로 복사-붙여넣기하여 통합
- 데이터 통합을 포기하고 각 시스템을 따로 사용
- 통합 솔루션을 도입하려 했으나 비용이 너무 비싸서 포기

---

## Job 5: 학생별 맞춤 관리가 필요한 케이스를 조기에 발견하고 싶다

### 상황 (Context)
- 재등록률을 높이기 위해서는 문제가 있는 학생을 조기에 발견하고 개입해야 함
- 출석률 하락, 학습 시간 감소, 모의고사 성적 하락 등의 신호를 포착해야 함
- 학생 수가 많으면 일일이 확인하기 어려워서 놓치는 경우가 발생

### Pain
- 학생별로 데이터를 일일이 확인하는데 시간이 많이 걸림
- 여러 지표(출석률, 학습 시간, 성적)를 종합적으로 봐야 하는데 수동으로는 어려움
- 문제가 있는 학생을 발견했을 때는 이미 늦은 경우가 많음 (예: 이미 결석이 1주일 지속됨)
- 어떤 학생에게 우선적으로 개입해야 할지 우선순위를 정하기 어려움

### Desired Outcome
- 대시보드에서 "주의 필요 학생" 리스트가 자동으로 생성됨 (출석률 하락, 학습 시간 감소, 성적 하락 등 기준)
- 학생별 위험도 점수가 자동 계산되어 우선순위가 명확함
- 문제 발생 시 즉시 알림(이메일/SMS)을 받아서 빠르게 대응 가능
- 학생별 맞춤 관리 이력과 결과를 추적하여 개입 효과를 측정 가능

### 현재 사용하는 대체 솔루션
- 주간 회의에서 교사들이 정성적으로 "이 학생이 이상하다"고 보고
- 엑셀에서 출석률이나 성적을 정렬해서 하위 학생을 확인 (수동)
- 문제가 명확히 드러난 후에야 발견 (예: 결석이 1주일 이상 지속된 후)
- 체계적인 모니터링 없이 직관에 의존

## Job 6: 원장/경영진에게 객관적 데이터 기반 경영 리포트를 제공하고 싶다

### 상황 (Context)
- 원장이나 경영진은 학원의 전반적인 경영 성과를 파악해야 함
- 월간/분기별로 재등록률, 매출, 학생 만족도 등을 리뷰해야 함
- 객관적 데이터를 바탕으로 경영 의사결정을 내려야 함

### Pain
- 경영 리포트를 만들기 위해 여러 부서에서 데이터를 수집하고 취합하는데 시간이 오래 걸림
- 데이터가 정확하지 않거나 최신이 아닐 수 있어서 신뢰도가 떨어짐
- 리포트 형식이 매번 달라져서 비교 분석이 어려움
- 경영 지표와 운영 지표를 연계하여 분석하기 어려움

### Desired Outcome
- 경영 대시보드에서 핵심 지표(재등록률, 매출, 학생 수, NPS 등)를 실시간으로 확인 가능
- 월간/분기별 리포트가 자동으로 생성되어 경영 회의 자료로 활용 가능
- 전년 동기 대비, 목표 대비 성과를 자동으로 비교하여 표시
- 지점별, 반별 성과를 경영 지표와 연계하여 분석 가능 (예: "A 지점의 재등록률이 높은 이유는 출석률이 높기 때문")

### 현재 사용하는 대체 솔루션
- 회계 시스템, 학생 관리 시스템에서 데이터를 수동으로 취합하여 엑셀로 리포트 작성
- 각 지점 관리자에게 데이터를 요청받아서 취합 (시간 지연, 데이터 불일치)
- 경영 회의에서 정성적 논의 위주, 데이터 기반 의사결정 부족
- 외부 컨설팅 업체에 리포트 제작을 의뢰 (비용 부담, 시간 지연)

---

## Job 7: 모의고사 성적 추이와 학습 활동 데이터를 연계하여 분석하고 싶다

### 상황 (Context)
- 모의고사 성적이 오르거나 내린 원인을 파악하기 위해 학습 활동 데이터를 함께 봐야 함
- "이 학생은 모의고사 성적이 올랐는데, 어떤 학습 활동이 도움이 됐을까?" 같은 질문에 답하고 싶음
- 학습 시간, 과제 완료도, 출석률 등이 모의고사 성적과 어떤 상관관계가 있는지 분석하고 싶음

### Pain
- 모의고사 플랫폼과 LMS가 별도 시스템이라서 데이터를 연계 분석하기 어려움
- 엑셀로 두 데이터를 합치려면 수동으로 작업해야 하고, 시간이 오래 걸림
- 상관관계를 시각화하거나 통계적으로 분석하는 것이 어려움
- "학습 시간이 많으면 성적이 오른다" 같은 가설을 검증하기 어려움

### Desired Outcome
- 모의고사 성적 추이와 학습 시간, 출석률, 과제 완료도 등을 한 차트에서 함께 확인 가능
- "학습 시간이 10시간 증가하면 모의고사 성적이 평균 5점 상승한다" 같은 인사이트가 자동으로 제시됨
- 학생별로 "이 학생은 학습 시간은 많지만 성적이 오르지 않는 패턴" 같은 이상 징후를 자동으로 감지
- 반별/지점별로 학습 활동과 성적의 상관관계를 비교 분석 가능

### 현재 사용하는 대체 솔루션
- 모의고사 성적과 학습 시간을 엑셀에 수동으로 입력하여 간단한 차트로 비교
- 두 시스템을 따로 보고 직관적으로 연관성을 파악 (정량적 분석 부족)
- 데이터 연계 분석을 포기하고 각각 따로 분석
