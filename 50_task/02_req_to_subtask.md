# REQ → Sub-Task 분해

본 문서는 SRS v3.0 Final의 모든 REQ를 입력(Input), 처리(Process), 출력(Output), 예외(Exception), 설정(Configuration), 테스트(Test) 항목으로 분해한 것입니다.

---

## Functional Requirements

### REQ-FUNC-001: 학생 검색

| Sub-Task | 설명 |
| --- | --- |
| **입력** | - 검색 쿼리 (학생 이름 또는 학생 ID, 부분 일치 지원)<br/>- 페이지네이션 파라미터 (page, limit)<br/>- JWT 토큰 (인증용) |
| **처리** | - Firestore `students` 컬렉션에서 검색 쿼리로 필터링<br/>- 이름 또는 ID로 부분 일치 검색 수행<br/>- 검색 결과 최대 50명 제한<br/>- 페이지네이션 적용 (page, limit) |
| **출력** | - 학생 목록 배열 (studentId, name, classId 등)<br/>- 전체 검색 결과 수 (total)<br/>- 현재 페이지 정보 (page) |
| **예외** | - 검색 쿼리가 비어있는 경우: 빈 목록 반환<br/>- Firestore 연결 실패: 에러 메시지 반환<br/>- 인증 토큰 만료/무효: 401 Unauthorized |
| **설정** | - Firestore 인덱스: `students` 컬렉션의 `name`, `studentId` 필드 인덱싱<br/>- 검색 결과 최대 개수: 50명<br/>- 페이지네이션 기본값: page=1, limit=20 |
| **테스트** | - 검색 기능 테스트: 정확한 이름/ID 검색<br/>- 부분 일치 검색 테스트: 부분 문자열 검색<br/>- 페이지네이션 테스트: 여러 페이지 조회<br/>- 빈 검색 결과 테스트 |

### REQ-FUNC-002: 리포트 생성 요청

| Sub-Task | 설명 |
| --- | --- |
| **입력** | - 학생 ID (studentId)<br/>- 리포트 형식 (format: "pdf")<br/>- JWT 토큰 (인증용) |
| **처리** | - 리포트 생성 요청 수신<br/>- 리포트 생성 프로세스 시작<br/>- 진행 상태 표시 (processing 상태) |
| **출력** | - 리포트 생성 요청 응답 (reportId, status: "processing")<br/>- 진행 상태 정보 |
| **예외** | - 학생 ID가 존재하지 않는 경우: 404 Not Found<br/>- 이미 진행 중인 리포트 생성 요청: 중복 요청 에러<br/>- Vercel 함수 실행 시간 제한 초과: 타임아웃 에러 |
| **설정** | - Vercel 서버리스 함수 실행 시간 제한: Hobby 10초 / Pro 60초<br/>- 비동기 처리 큐 시스템 설정 (필요 시) |
| **테스트** | - 리포트 생성 요청 API 테스트<br/>- 진행 상태 응답 테스트<br/>- 중복 요청 처리 테스트 |

### REQ-FUNC-003: 출석 데이터 조회

| Sub-Task | 설명 |
| --- | --- |
| **입력** | - 학생 ID (studentId)<br/>- 기간 정보 (전체 기간, 최근 4주) |
| **처리** | - Firestore `attendance` 컬렉션에서 학생 ID로 조회<br/>- 날짜 기준 정렬 (내림차순)<br/>- 전체 기간 출석률 계산: (출석 일수 / 전체 수업 일수) × 100<br/>- 최근 4주 출석률 계산 |
| **출력** | - 출석 데이터 배열 (date, isPresent)<br/>- 전체 기간 출석률 (%)
<br/>- 최근 4주 출석률 (%) |
| **예외** | - 학생 ID가 존재하지 않는 경우: 빈 배열 반환<br/>- 출석 데이터가 없는 경우: 출석률 0% 반환<br/>- Firestore 쿼리 실패: 에러 메시지 반환 |
| **설정** | - Firestore 인덱스: `attendance` 컬렉션의 `studentId` + `date` (내림차순)<br/>- 최근 4주 계산 기준: 현재 날짜 기준 역산 |
| **테스트** | - 출석 데이터 조회 테스트<br/>- 출석률 계산 로직 테스트<br/>- 최근 4주 필터링 테스트 |

### REQ-FUNC-004: 학습 시간 데이터 조회

| Sub-Task | 설명 |
| --- | --- |
| **입력** | - 학생 ID (studentId)<br/>- 목표 학습 시간 (설정값 또는 기본값) |
| **처리** | - Firestore `study_time` 컬렉션에서 학생 ID로 조회<br/>- 날짜 기준 정렬 (내림차순)<br/>- 일평균 학습 시간 계산: (전체 학습 시간 합계 / 학습 일수)<br/>- 주평균 학습 시간 계산: (전체 학습 시간 합계 / 주 수)<br/>- 목표 대비 달성률 계산: (실제 평균 / 목표) × 100 |
| **출력** | - 학습 시간 데이터 배열 (date, hours)<br/>- 일평균 학습 시간 (시간)<br/>- 주평균 학습 시간 (시간)<br/>- 목표 대비 달성률 (%) |
| **예외** | - 학생 ID가 존재하지 않는 경우: 빈 배열 반환<br/>- 학습 시간 데이터가 없는 경우: 평균 0, 달성률 0% 반환<br/>- Firestore 쿼리 실패: 에러 메시지 반환 |
| **설정** | - Firestore 인덱스: `study_time` 컬렉션의 `studentId` + `date` (내림차순)<br/>- 목표 학습 시간 기본값 (설정 필요) |
| **테스트** | - 학습 시간 데이터 조회 테스트<br/>- 집계 계산 테스트 (일평균, 주평균)<br/>- 목표 대비 달성률 계산 테스트 |

### REQ-FUNC-005: 모의고사 성적 데이터 조회

| Sub-Task | 설명 |
| --- | --- |
| **입력** | - 학생 ID (studentId) |
| **처리** | - Firestore `mock_exam` 컬렉션에서 학생 ID로 조회<br/>- 시험 날짜 기준 정렬 (내림차순)<br/>- 최근 5회 모의고사 성적 필터링<br/>- 추이 분석: 점수 변화 추세 계산<br/>- 등급 변화 분석: 등급 상승/하락 판단 |
| **출력** | - 최근 5회 모의고사 성적 배열 (examRound, score, grade, examDate)<br/>- 추이 분석 결과 (상승/하락/유지)<br/>- 등급 변화 정보 |
| **예외** | - 학생 ID가 존재하지 않는 경우: 빈 배열 반환<br/>- 모의고사 데이터가 5회 미만인 경우: 존재하는 데이터만 반환<br/>- Firestore 쿼리 실패: 에러 메시지 반환 |
| **설정** | - Firestore 인덱스: `mock_exam` 컬렉션의 `studentId` + `examDate` (내림차순)<br/>- 최근 회차 수: 5회 |
| **테스트** | - 모의고사 성적 조회 테스트<br/>- 추이 분석 테스트<br/>- 등급 변화 분석 테스트 |
