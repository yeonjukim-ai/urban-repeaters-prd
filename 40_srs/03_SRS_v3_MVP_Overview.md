## 1. 변경 사항 요약

본 SRS v3.0은 SRS v2.0을 MVP 기준에 맞게 조정한 버전입니다.

### 주요 변경 사항

1. **기술 스택 단순화**
   - Python FastAPI + LangChain 제거 → Java Spring Boot 내 규칙 엔진
   - Kubernetes 제거 → Docker Compose / 단일 서버
   - gRPC 제거 → REST API만 사용
   - OpenTelemetry + APM 제거 → 기본 로깅만

2. **기능 범위 축소**
   - 정기 리포트 자동 전송 → 수동 이메일 전송 버튼
   - 위험 신호 자동 감지 → 제거 (Post-MVP)
   - 데이터 자동 동기화 → 수동 업로드만
   - 외부 시스템 API 연동 → CSV/Excel 업로드만

3. **비기능 요구사항 완화**
   - 시스템 가동률: 99% → 95%
   - 동시 사용자: 100명 → 20명
   - 리포트 생성 성공률: 95% → 90%

## 2. Scope (MVP 범위)

### 2.1 In-Scope (MVP v3)

#### **1. 학생별 종합 성과 리포트 즉시 생성 (F1 - 단순화)**

**핵심 기능**:
- 학생 검색 (이름/ID)
- 리포트 생성 요청
- 출석률, 학습 시간, 모의고사 성적, 과제 완료도 데이터 조회
- **PDF 형식만** 리포트 생성 (30초 이내)
- **템플릿 기반 인사이트 생성** (최대 3개, LLM 없이)
- 리포트 다운로드
- 리포트 생성 이력 저장

**단순화 사항**:
- PNG 형식 제외
- 반 평균 데이터는 더미 데이터 또는 하드코딩
- 인사이트는 템플릿 기반 규칙 엔진 사용 (LLM 제외)

#### **2. 기본 데이터 통합 (F2 - 단순화)**

**핵심 기능**:
- **CSV/Excel 파일 업로드만** 지원 (API 연동 제외)
- 수동 데이터 입력 인터페이스
- 데이터 검증 (형식, 범위)
- 데이터 통합 (학생 ID 기준)
- **단순 통합 대시보드** (테이블 형태, 그래프 제외)

**단순화 사항**:
- 외부 시스템 API 연동 제외
- 데이터 자동 동기화 제외
- 데이터 불일치 자동 감지 제외
- 대시보드는 테이블 형태만 (그래프/차트 제외)

#### **3. 수동 리포트 전송 (F3 - 단순화)**

**핵심 기능**:
- 리포트 생성 후 **수동 이메일 전송 버튼** 제공
- 이메일 첨부 전송 (PDF)
- 리포트 전송 이력 저장 (기본)

**단순화 사항**:
- 자동 스케줄링 제외
- SMS 전송 제외
- 리포트 열람 여부 추적 제외
- 개인화 메시지 자동 생성 제외

#### **4. 기본 사용자 관리**

**핵심 기능**:
- 사용자 로그인/로그아웃
- JWT 토큰 기반 인증
- **단일 관리자 역할** (역할 구분 제외)

**단순화 사항**:
- 역할 기반 접근 제어 제외 (Post-MVP)

#### **5. 기본 시스템 설정**

**핵심 기능**:
- 리포트 템플릿 기본 설정 (커스터마이징 제외)

**단순화 사항**:
- 학원 로고, 색상 커스터마이징 제외 (Post-MVP)

### 2.2 Out-of-Scope (Post-MVP)

#### **자동화 기능**
1. 정기 리포트 자동 전송 (스케줄 기반)
2. 위험 신호 자동 감지 및 알림
3. 데이터 불일치 자동 감지
4. 외부 시스템 API 자동 동기화

#### **고급 기능**
1. PNG 리포트 생성
2. 역할 기반 접근 제어 (다중 역할)
3. 리포트 템플릿 고급 커스터마이징
4. SMS 전송
5. 리포트 열람 여부 추적

#### **인프라/운영**
1. Kubernetes 배포
2. OpenTelemetry + APM
3. gRPC 통신
4. LLM 기반 인사이트 생성
