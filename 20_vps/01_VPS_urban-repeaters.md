- **시장 개요 & TAM/SAM/SOM** (`10_analysis/11_market-overview_TAM-SAM-SOM.md`): 시장 정의, 규모 가설, 세그먼트 정의

### 작성 기준

  - 상위 레벨에서는 입시 관련 학원 전체를 대상으로 TAM/SAM을 추정하되,  
  - VPS·MVP 설계에서는 그중 **도심형 통학 관리형 재수 학원 약 300~500개소(학생 수 150~250명/학원)를 핵심 타깃 풀**로 본다.
- **핵심 포지셔닝**: 학부모 커뮤니케이션 효율화와 데이터 기반 의사결정 지원 도구
- **타깃 우선순위**: 중형 도심형 통학 관리형 학원(2~5개 지점, 학생 수 200~500명) → 소형 단일 지점 학원 → 프리미엄 소규모 학원

## 2. 핵심 Job to be Done 정리

### 우선순위별 JTBD

#### Priority 1 (MVP 필수)
1. **학부모 상담 전에 학생의 종합 성과 리포트를 빠르게 준비하고 싶다**
   - 상담 30분 전 버튼 클릭으로 자동 생성, 출석률/학습시간/성적 추이/과제 완료도 통합 표시
   - 현재: 엑셀 템플릿 + 수동 입력, 주 5~6시간 소비 → 목표: 주 1시간 이하

2. **여러 시스템의 데이터를 한 곳에서 통합하여 확인하고 싶다**
   - 출석 앱, LMS, 모의고사 플랫폼, 결제 시스템 데이터를 단일 대시보드에서 통합 확인
   - 현재: 여러 시스템 오가며 복사-붙여넣기 → 목표: API 자동 연동

#### Priority 2 (초기 런칭)
3. **학부모에게 정기적으로 전문적이고 일관된 리포트를 자동으로 전달하고 싶다**
   - 스케줄 설정 시 자동 생성 및 이메일/SMS 전송, 전송 내역 및 열람 여부 추적
   - 현재: 주 10~20시간 리포트 제작 + 전송 → 목표: 자동화로 시간 제로

4. **학생별 맞춤 관리가 필요한 케이스를 조기에 발견하고 싶다**
   - 출석률 하락, 학습 시간 감소, 성적 하락 등 위험 신호 자동 감지 및 알림
   - 현재: 수동 확인으로 놓치는 경우 발생 → 목표: 실시간 자동 모니터링

#### Priority 3 (확장)
5. **지점별/반별 성과를 비교하여 개선점을 도출하고 싶다**
   - 대시보드에서 지점/반별 핵심 지표 비교, 시각적 차트 및 인사이트 자동 제시

6. **원장/경영진에게 객관적 데이터 기반 경영 리포트를 제공하고 싶다**
   - 경영 대시보드에서 재등록률, 매출, NPS 등 핵심 지표 실시간 확인

7. **모의고사 성적 추이와 학습 활동 데이터를 연계하여 분석하고 싶다**
   - 학습 시간, 출석률과 모의고사 성적의 상관관계 분석 및 인사이트 제시

## 3. 핵심 Value Proposition 요약 (3~5줄)

**도심형 통학 관리형 재수 학원을 위한 성과 리포트 자동화 SaaS**

주 10~20시간 소요되던 리포트 제작을 자동화하여 전략적 업무에 집중할 시간을 확보하고, 여러 시스템에 분산된 데이터를 단일 대시보드로 통합하여 실시간 모니터링과 분석이 가능하도록 한다. 특히 학부모 상담 시 30초 내 종합 성과 리포트를 생성하여 전문성을 높이고, 정기적으로 일관된 리포트를 자동 전송함으로써 학부모 만족도와 재등록률 향상에 기여한다. 단순 리포트 도구를 넘어서 데이터 기반 의사결정과 학원 운영 효율화를 지원하는 전략적 경영 도구로 포지셔닝한다.

## 5. 기능 아이디어(F1~F?) 목록

### F1. 학생별 종합 성과 리포트 즉시 생성 (MVP: Yes, Priority: High)
- **기능 설명**: 학부모 상담 전 해당 학생의 최신 출석률, 학습 시간, 모의고사 성적 추이, 과제 완료도, 반 평균 대비 위치 등을 한 페이지 리포트로 자동 생성. 리포트 생성 버튼 클릭 후 30초 이내 PDF/이미지 형태로 제공.
- **해결하는 페인**: 
  - 여러 시스템을 오가며 데이터 수집에 시간 소비 (JTBD #1)
  - 엑셀 템플릿 복사 및 수동 입력의 번거로움
  - 상담 직전 급하게 리포트 제작 시 실수 발생
- **기대 효과**: 
  - 리포트 준비 시간 주 5~6시간 → 주 1시간 이하로 단축
  - 리포트 일관성 및 전문성 향상으로 학부모 신뢰도 증가
  - 상담 효율성 향상으로 상담 시간 단축

---

### F2. 다중 시스템 데이터 자동 통합 (MVP: Yes, Priority: High)
- **기능 설명**: 출석 관리 앱, LMS, 모의고사 플랫폼, 결제 시스템 등 여러 시스템의 데이터를 API 연동 또는 파일 업로드 방식으로 자동 통합. 단일 대시보드에서 모든 데이터를 실시간으로 확인 가능. 데이터 불일치 시 자동 알림.
- **해결하는 페인**: 
  - 여러 시스템 간 데이터 수동 통합의 번거로움 (JTBD #4)
  - 시스템 간 데이터 불일치 문제
  - 각 시스템마다 다른 인터페이스로 인한 비효율
- **기대 효과**: 
  - 데이터 통합 시간 80% 이상 단축
  - 데이터 정확성 향상으로 리포트 신뢰도 증가
  - 시스템 간 데이터 불일치 조기 감지 및 해결

---

### F3. 학부모 정기 리포트 자동 전송 (MVP: Yes, Priority: High)
- **기능 설명**: 주 1회 또는 월 2회 등 스케줄 설정 시 학생별 리포트 자동 생성 및 이메일/SMS 전송. 리포트 열람 여부 추적 및 전송 내역 관리. 리포트 템플릿 커스터마이징 가능 (학원 로고, 색상 등).
- **해결하는 페인**: 
  - 정기 리포트 제작 및 전송에 주 10~20시간 소비 (JTBD #3)
  - 리포트 형식 불일치로 인한 전문성 저하
  - 리포트 전송 누락 시 학부모 불만 발생
- **기대 효과**: 
  - 리포트 전송 업무 시간 주 15시간 → 0시간 (완전 자동화)
  - 일관된 리포트로 학원 브랜드 이미지 향상
  - 학부모 만족도 및 재등록률 향상 기여

---

### F4. 학생별 위험 신호 자동 감지 및 알림 (MVP: Yes, Priority: High)
- **기능 설명**: 출석률 하락, 학습 시간 감소, 모의고사 성적 하락 등 위험 신호를 자동으로 감지하여 "주의 필요 학생" 리스트 자동 생성. 학생별 위험도 점수 계산 및 우선순위 제시. 위험 신호 발생 시 즉시 알림(이메일/SMS) 전송.
- **해결하는 페인**: 
  - 학생별 데이터 수동 확인의 비효율 (JTBD #5)
  - 문제 학생 조기 발견의 어려움
  - 우선순위 설정의 애매함
- **기대 효과**: 
  - 문제 학생 조기 발견으로 개입 시점 선제적 대응 가능
  - 학생 이탈률 감소 및 재등록률 향상
  - 학사 관리자의 모니터링 효율성 대폭 향상

---

### F5. 지점별/반별 성과 비교 대시보드 (MVP: No, Priority: Mid)
- **기능 설명**: 지점별, 반별 핵심 지표(재등록률, 모의고사 평균, 출석률 등)를 대시보드에서 시각적으로 비교. 바 차트, 라인 차트 등으로 성과 격차 직관적 표시. 자동 인사이트 제시 (예: "이번 달 B 지점의 출석률이 평균 대비 15% 낮습니다").
- **해결하는 페인**: 
  - 지점/반별 데이터 수동 취합 및 비교의 번거로움 (JTBD #2)
  - 엑셀로 차트 작성의 어려움
  - 객관적 데이터 기반 의사결정의 어려움
- **기대 효과**: 
  - 경영 회의 시간 단축 및 의사결정 효율성 향상
  - 지점/반별 성과 격차 객관적 파악으로 개선점 도출 용이

---

### F6. 경영 대시보드 (MVP: No, Priority: Mid)
- **기능 설명**: 원장/경영진용 대시보드에서 핵심 경영 지표(재등록률, 매출, 학생 수, NPS 등) 실시간 확인. 월간/분기별 경영 리포트 자동 생성. 전년 동기 대비, 목표 대비 성과 자동 비교.
- **해결하는 페인**: 
  - 경영 리포트 제작을 위한 데이터 수집 시간 소비 (JTBD #6)
  - 데이터 정확성 및 최신성 문제
  - 경영 지표와 운영 지표 연계 분석의 어려움
- **기대 효과**: 
  - 경영 의사결정 속도 향상
  - 데이터 기반 경영으로 수익성 개선 기여

---

### F7. 학습 활동과 성적 상관관계 분석 (MVP: No, Priority: Low)
- **기능 설명**: 모의고사 성적 추이와 학습 시간, 출석률, 과제 완료도 등을 한 차트에서 함께 시각화. 상관관계 분석 결과 및 인사이트 자동 제시 (예: "학습 시간이 10시간 증가하면 모의고사 성적이 평균 5점 상승").
- **해결하는 페인**: 
  - 모의고사 플랫폼과 LMS 데이터 연계 분석의 어려움 (JTBD #7)
  - 상관관계 시각화 및 통계 분석의 어려움
- **기대 효과**: 
  - 학습 전략 효과성 검증 가능
  - 데이터 기반 학습 가이드 제공으로 성과 향상 기여

---

### F8. 리포트 템플릿 커스터마이징 (MVP: No, Priority: Mid)
- **기능 설명**: 리포트 디자인, 레이아웃, 학원 로고, 색상 등을 커스터마이징하여 학원 브랜드에 맞게 리포트 생성. 여러 템플릿 선택 가능.
- **해결하는 페인**: 
  - 리포트 형식이 학원 브랜드와 어울리지 않음
  - 리포트 전문성 및 차별화 부족
- **기대 효과**: 
  - 학원 브랜드 이미지 강화
  - 경쟁 학원 대비 차별화된 리포트 서비스 제공

---

### F9. 모바일 앱 (학부모용) (MVP: No, Priority: Low)
- **기능 설명**: 학부모용 모바일 앱에서 자녀의 학습 현황 리포트를 실시간으로 확인. 푸시 알림으로 리포트 수신 알림.
- **해결하는 페인**: 
  - 학부모가 리포트를 이메일로만 받아 확인이 불편함
  - 리포트 열람 여부 추적의 어려움
- **기대 효과**: 
  - 학부모 접근성 향상 및 만족도 증가
  - 리포트 활용도 향상으로 학원-학부모 커뮤니케이션 강화

## 6. 경쟁/대체 솔루션 (Competitors & Alternatives)

### 6.1 직접 경쟁 솔루션
- **없음 (명확한 직접 경쟁 없음)**: 도심형 통학 관리형 재수 학원 시장에 특화된 성과 리포트 SaaS는 현재 시장에 존재하지 않음.

### 6.2 대체 솔루션

#### A. 엑셀/구글 시트 기반 수동 제작 (현재 가장 일반적)
- **특징**: 
  - 학원 내부 직원이 엑셀 템플릿을 만들고 수동으로 데이터 입력
  - 주 10~20시간 소요, 실수 빈발, 일관성 부족
- **우리 대비 강점**: 
  - 자동화로 시간 90% 이상 절감
  - 데이터 정확성 및 일관성 보장
  - 전문적 디자인 리포트
- **우리 대비 약점**: 
  - 초기 도입 비용 발생 (구독료)
  - 학습 곡선 존재

#### B. 대형 학원 관리 시스템(AMS)의 리포트 기능
- **특징**: 
  - 메가스터디, 이투스 등 대형 학원 체인용 통합 관리 시스템
  - 과도한 기능과 높은 가격 (월 수백만원 이상)
- **우리 대비 강점**: 
  - 가격 경쟁력 (월 50~150만원)
  - 도심형 중소 학원에 맞춤화된 기능
  - 도입 및 운영 용이성
- **우리 대비 약점**: 
  - 대형 시스템의 브랜드 신뢰도
  - 다양한 기능의 통합성

#### C. 범용 BI 도구 (Tableau, Power BI 등)
- **특징**: 
  - 기업용 데이터 시각화 도구
  - 교육 도메인 특화 기능 부족, 학습 곡선 높음
- **우리 대비 강점**: 
  - 교육 도메인 특화 기능 (학생별 리포트, 학부모 전송 등)
  - 직관적 인터페이스 및 낮은 학습 곡선
  - 도입 지원 서비스 제공
- **우리 대비 약점**: 
  - 대형 BI 도구의 브랜드 인지도
  - 고급 분석 기능

#### D. 외부 리포트 제작 아웃소싱
- **특징**: 
  - 외부 업체에 리포트 제작을 의뢰
  - 비용 부담 (월 수십만원~수백만원), 시간 지연 발생
- **우리 대비 강점**: 
  - 실시간 리포트 생성 가능
  - 낮은 비용 (자동화로 인건비 절감)
  - 즉각적인 리포트 수정 및 커스터마이징 가능
- **우리 대비 약점**: 
  - 외부 업체의 전문성 (특정 업체 한정)

#### E. 각 시스템별 리포트 기능 활용
- **특징**: 
  - 출석 앱, LMS, 모의고사 플랫폼 각각의 리포트 기능을 따로 사용
  - 데이터 통합이 어려움
- **우리 대비 강점**: 
  - 다중 시스템 데이터 통합 제공
  - 종합적인 리포트 생성
  - 데이터 연계 분석 가능
- **우리 대비 약점**: 
  - 각 시스템의 리포트 기능은 무료 또는 저렴

### 6.3 경쟁 우위 요소
1. **니치 시장 특화**: 도심형 통학 관리형 재수 학원 시장에 특화된 유일한 솔루션
2. **명확한 ROI**: 리포트 제작 시간 절감으로 즉각적인 비용 절감 체감 가능
3. **높은 전환 비용**: 데이터 통합 및 리포트 자동화 시스템으로 전환 후 교체 어려움
4. **전략적 가치**: 단순 리포트 도구를 넘어 재등록률 향상에 기여하는 경영 도구로 포지셔닝

### 7.3 비즈니스 임팩트 (파일럿 고객 기준)

#### 시간 절감 효과
- **리포트 제작 시간**: 주 15시간 → 주 1시간 이하 (약 93% 절감)
- **데이터 통합 시간**: 주 5시간 → 주 30분 이하 (약 90% 절감)
- **학부모 상담 준비 시간**: 상담당 1시간 → 상담당 5분 이하 (약 92% 절감)
- **연간 시간 절감**: 학원당 연간 약 700시간 절감 (주 15시간 × 46주)

#### 비용 절감 효과
- **인건비 절감**: 학원당 연간 약 350만원 절감 (시간당 5천원 가정 시)
- **아웃소싱 비용 절감**: 월 리포트 제작 외주 비용 제로화 (월 50만원 가정 시, 연간 600만원)

#### 운영 효율성 향상
- **리포트 정확도**: 85% → 99% 이상 (데이터 오류 감소)
- **학부모 만족도**: NPS 50 → 65 이상 (전문적 리포트 제공)
- **재등록률**: 현재 70% → 75% 이상 (학부모 커뮤니케이션 강화)

#### 수익성 개선
- **학원당 월 구독료**: 50~150만원 (학생 수 및 기능에 따라 차등)
- **파일럿 고객 3개 학원 기준 연간 MRR**: 약 300만원 (평균 100만원/월/학원)
- **목표 ARR (90일 후)**: 약 1,200만원 (파일럿 3개 × 100만원 × 4개월)

### 7.4 단계별 기대 임팩트

#### 30일 (MVP 개발 완료)
- **기능 개발**: F1, F2, F3 기본 기능 완성

## 8. 핵심 가설, 리스크, 검증 방법

### 8.1 핵심 가설 (Key Hypotheses)

#### 가설 1: 시간 절감 가치 인식
- **내용**: 학원들이 리포트 제작에 주 10~20시간을 소비하고 있으며, 이를 자동화할 의사와 예산이 있다.
- **중요도**: 매우 높음 (핵심 Value Proposition)
- **검증 방법**: 
  - 파일럿 고객 인터뷰에서 리포트 제작 시간 측정
  - 파일럿 사용 전후 시간 비교 (Before/After 분석)
  - 파일럿 고객의 월 구독료 지불 의사 확인

#### 가설 2: 다중 시스템 데이터 통합 니즈
- **내용**: 학원들이 출석 앱, LMS, 모의고사 플랫폼 등 여러 시스템을 사용하며, 통합이 어렵다고 느끼고 있다.
- **중요도**: 매우 높음 (차별화 요소)
- **검증 방법**: 
  - 파일럿 고객의 현재 사용 시스템 조사
  - API 연동 가능 여부 확인
  - 데이터 통합 후 만족도 측정

#### 가설 3: 학부모 커뮤니케이션의 전략적 가치
- **내용**: 전문적이고 일관된 리포트 제공이 학부모 만족도와 재등록률 향상에 기여한다.
- **중요도**: 높음 (장기적 가치)
- **검증 방법**: 
  - 파일럿 고객의 학부모 만족도 조사 (Before/After)
  - 재등록률 추이 분석 (6개월~1년 단위)
  - 리포트 전송 빈도와 재등록률 상관관계 분석

#### 가설 4: 가격 민감도
- **내용**: 월 50~150만원의 구독료는 학원들이 지불할 수 있는 수준이며, ROI를 체감할 수 있다.
- **중요도**: 높음 (비즈니스 모델)
- **검증 방법**: 
  - 다양한 가격대 테스트 (Price Sensitivity Analysis)
  - 파일럿 고객의 구독료 지불 의사 확인
  - 시간 절감 효과를 비용으로 환산하여 ROI 계산

#### 가설 5: 시장 규모
- **내용**: 도심형 통학 관리형 재수 학원 300~500개소 중 10%인 30~50개 학원이 초기 3년 내 고객이 될 수 있다.
- **중요도**: 높음 (시장 기회)
- **검증 방법**: 
  - 시장 조사: 실제 도심형 학원 수 확인
  - 파일럿 고객 확보 속도 측정
  - 확보된 고객의 추천 의향 (Referral Rate) 측정

### 8.2 핵심 리스크 (Key Risks)

#### 리스크 1: 시스템 연동 어려움
- **내용**: 출석 앱, LMS, 모의고사 플랫폼 등 각 시스템의 API가 없거나 연동이 어려울 수 있다.
- **영향도**: 매우 높음 (핵심 기능 불가)
- **완화 방안**: 
  - 파일럿 전에 주요 시스템의 API 연동 가능 여부 사전 조사
  - API가 없는 경우 파일 업로드 방식(CSV, Excel) 대안 제공
  - 주요 시스템과의 파트너십 모색

#### 리스크 2: 고객 확보 어려움
- **내용**: 학원들이 새로운 도구 도입에 보수적이거나, 학습 곡선을 우려하여 도입을 꺼릴 수 있다.
- **영향도**: 높음 (비즈니스 성장 저해)
- **완화 방안**: 
  - 파일럿 프로그램으로 리스크 부담 완화 (무료 또는 저가)
  - 상세한 온보딩 및 교육 프로그램 제공
  - 기존 고객의 성공 사례 및 추천 활용

#### 리스크 3: 가격 경쟁력 부족
- **내용**: 기존 엑셀 방식의 무료성 대비 월 50~150만원 구독료가 부담될 수 있다.
- **영향도**: 중간 (시장 진입 장벽)
- **완화 방안**: 
  - 명확한 ROI 제시 (시간 절감 효과를 비용으로 환산)
  - 학생 수 기반 차등 가격 제공 (소형 학원 저가 플랜)
  - 무료 체험 기간 제공 (14일 또는 30일)

#### 리스크 4: 기능 복잡도 및 학습 곡선
- **내용**: 사용자들이 도구를 배우는 데 시간이 오래 걸리거나, 직관적이지 않아 포기할 수 있다.
- **영향도**: 중간 (사용자 만족도 저하)
- **완화 방안**: 
  - 최대한 직관적인 UI/UX 설계
  - 단계별 온보딩 튜토리얼 제공
  - 실시간 고객 지원 채널 운영 (채팅, 전화)

#### 리스크 5: 대형 경쟁자 진입
- **내용**: 대형 학원 관리 시스템 업체나 교육 플랫폼 업체가 유사한 기능을 빠르게 개발하여 시장에 진입할 수 있다.
- **영향도**: 중간~높음 (장기적 위협)
- **완화 방안**: 
  - 빠른 시장 진입 및 고객 확보로 선점 우위 구축
  - 고객 데이터 및 통합 연동을 통한 전환 비용(Lock-in) 증가
  - 니치 시장 특화 기능 개발로 차별화

#### 리스크 6: 시장 규모 과대 추정
- **내용**: 실제 도심형 통학 관리형 재수 학원 수나 리포트 자동화 니즈가 추정보다 적을 수 있다.
- **영향도**: 높음 (비즈니스 모델 위협)
- **완화 방안**: 
  - 실제 시장 조사 실시 (학원 수, 학생 수, 리포트 제작 현황)
  - 파일럿 고객 인터뷰를 통한 니즈 검증
  - 인접 시장 확장 가능성 검토 (일반 고등학교 입시 학원 등)

### 8.3 검증 방법 (Validation Methods)

#### 검증 1: 파일럿 프로그램
- **목적**: 실제 고객 환경에서 제품 가치 검증
- **방법**: 
  - 3~5개 학원과 90일 파일럿 진행
  - Before/After 비교: 리포트 제작 시간, 만족도, 재등록률 등
  - 정량적 데이터 수집: 리포트 생성 횟수, 사용 빈도, 시간 절감 등
  - 정성적 피드백 수집: 인터뷰, 설문 조사
- **성공 기준**: 
  - 80% 이상 파일럿 고객이 정식 구독 전환
  - 리포트 제작 시간 80% 이상 절감
  - NPS 50 이상

#### 검증 2: 고객 인터뷰
- **목적**: JTBD 및 Pain Point 검증
- **방법**:

- **성공 기준**: 
  - JTBD 가설 80% 이상 일치
  - 핵심 Pain Point 재확인
  - 기능 우선순위 합의

#### 검증 3: 가격 민감도 테스트
- **목적**: 적정 가격대 및 가격 민감도 파악
- **방법**:

- **성공 기준**:

#### 검증 4: 시스템 연동 사전 조사
- **목적**: 데이터 통합 기술적 타당성 검증
