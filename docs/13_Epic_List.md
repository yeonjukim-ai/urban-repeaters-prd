## Epic 간 의존성 관계

```
E0 (프로젝트 기반 설정)
  ↓
E1 (인증) + E2 (DB/Storage)
  ↓
E3 (학생 관리) ──┐
  ↓              │
E4 (데이터 통합) ─┼──→ E5 (리포트 생성)
  ↓              │         ↓
E7 (대시보드)    │    E6 (리포트 전송)
                │         ↑
E8 (템플릿) ─────┘
  ↓
E9 (NFR)
```

## Epic 우선순위 설명

### P0 (Must Have) - MVP 출시 필수
- **E0, E1, E2**: 시스템 기반 인프라, 없으면 시스템이 동작하지 않음
- **E3**: 리포트 생성의 전제 조건 (학생 정보 필요)
- **E4**: 리포트 생성의 전제 조건 (데이터 없으면 리포트 생성 불가)
- **E5**: 핵심 기능 (리포트 생성)
- **E8**: 리포트 생성에 필수 (템플릿 없으면 리포트 생성 불가)

### P1 (Should Have) - MVP 출시 권장
- **E6**: 리포트 전송은 핵심 기능은 아니지만 사용자 경험에 중요
- **E7**: 통합 대시보드는 선택 기능이지만 데이터 확인에 유용

### P2 (Nice to Have) - MVP 이후 추가 가능
- **E9**: 일부는 P1 (성능, 보안), 일부는 P2 (모니터링, 최적화)
