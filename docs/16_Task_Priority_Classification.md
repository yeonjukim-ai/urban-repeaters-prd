# 태스크 우선순위 분류

**문서 ID**: TASK-PRIORITY-001  
**작성일**: 2025-01-27  
**버전**: 1.0  
**목적**: MVP 개발을 위한 태스크 우선순위 분류 및 조정

---

## 분류 기준

### 1. 반드시 MVP에 포함되어야 하는 태스크 (P0 - Must Have)
- 시스템이 동작하기 위한 최소 필수 기능
- 핵심 비즈니스 가치를 제공하는 기능
- 없으면 MVP가 완성되지 않는 태스크

### 2. "있으면 좋지만" 2차 버전으로 미뤄도 되는 태스크 (P1/P2 - Should Have / Nice to Have)
- 사용자 경험 향상 기능
- 선택적 기능
- MVP 이후 추가 가능한 기능

### 3. 이번 단계에서는 더미/가짜 데이터로 대체해도 되는 태스크 (DUMMY)
- 실제 구현 대신 하드코딩된 값이나 더미 데이터로 대체 가능
- MVP 단계에서 단순화하여 빠르게 개발 가능
- 이후 버전에서 실제 구현으로 교체

### 추가 고려사항

**더미 데이터로 대체 가능한 영역 (현재는 구현되어 있지만 단순화 가능):**

1. **초기 Seed 데이터**: 개발/테스트용 더미 학생 데이터
   - 현재 태스크 리스트에는 없지만, 필요 시 추가 가능
   - 실제 데이터 없이도 개발 가능하도록 더미 데이터 제공

2. **일부 통계 계산**: 복잡한 통계 계산은 MVP에서 단순화
   - 예: 성적 추이 분석을 단순 비교로 대체
   - 예: 목표 대비 달성률을 고정 목표값으로 대체

## MVP 개발 전략

### Phase 1: 핵심 인프라 (Week 1)
- E0, E1, E2의 P0 태스크만 진행
- 더미 데이터 없이 실제 구현

- E3, E4, E5, E8의 P0 태스크 진행

### Phase 3: 선택 기능 (Week 5)
- E6, E7의 P1 태스크 진행 (시간 여유 시)
- 또는 MVP 출시 준비

## 권장사항

1. **MVP 범위 축소**: E6 (리포트 전송)는 MVP 출시 후 즉시 추가 가능하도록 준비하되, MVP 출시는 리포트 생성까지만 완료해도 가능

2. **더미 데이터 활용**: T-API-055는 반드시 더미 데이터로 구현하여 개발 속도 향상

3. **점진적 개선**: P1 태스크는 MVP 출시 후 사용자 피드백을 받아 우선순위 재조정

4. **기술 부채 관리**: 더미 데이터 사용 시 주석으로 명확히 표시하고, 2차 버전에서 실제 구현으로 교체할 계획 수립
